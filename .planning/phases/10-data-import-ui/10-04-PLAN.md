  **2. Modify `frontend/src/components/actions/ActionsTab.tsx`:**

  The ActionsTab currently renders loading/error/empty states and then `<ActionsList>`. Modify it to always render the DataImportSection above the existing content:

  ```typescript
  import { DataImportSection } from './DataImportSection'

  export function ActionsTab() {
    const { data, isLoading, isError, error, refetch } = useActions()

    return (
      <div className="space-y-6">
        {/* Data import is always visible regardless of actions loading state */}
        <DataImportSection />

        {/* Existing actions section */}
        {isLoading ? (
          <div className="space-y-2">...</div>
        ) : isError ? (
          <ErrorAlert ... />
        ) : items.length === 0 ? (
          <EmptyState title="No pending actions" />
        ) : (
          <ActionsList items={items} />
        )}
      </div>
    )
  }
  ```

  Key: `DataImportSection` renders unconditionally — it's always available even while actions are loading or if they error out. The existing actions list code stays exactly as-is, just wrapped in the conditional block. The `items` variable assignment (`const items = data?.actions ?? []`) moves above the return statement.
