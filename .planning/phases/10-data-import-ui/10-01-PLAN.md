<task type="auto">
  <name>Task 2: Add Vite proxy for /ingestion and create useImportHistory hook</name>
  <files>frontend/vite.config.ts, frontend/src/hooks/useImportHistory.ts</files>
  <action>
  1. In `frontend/vite.config.ts`, add `/ingestion` to the `server.proxy` object alongside the existing `/api` and `/health` proxies:
     ```typescript
     '/ingestion': {
       target: 'http://localhost:8000',
       changeOrigin: true,
     },
     ```
     Place it after the `/api` entry and before the `/health` entry. This ensures the Vite dev server forwards all `/ingestion/*` requests to the FastAPI backend.

  2. Create `frontend/src/hooks/useImportHistory.ts`:
     - Define `ImportRun` interface with fields: `id: number`, `platform: string`, `filename: string`, `inserted_count: number`, `updated_count: number`, `skipped_count: number`, `imported_at: string` (ISO datetime)
     - Export a `useImportHistory(limit = 10)` function that returns `useQuery<ImportRun[]>(...)` with:
       - `queryKey: ['ingestion', 'history', limit]`
       - `queryFn`: use raw `fetch('/ingestion/history?limit=${limit}')` — NOT apiFetch (ingestion endpoints are NOT under /api/)
       - Throw on non-ok response (check `response.ok`)
       - `staleTime: 60_000` (1 minute)
     - Import `useQuery` from `@tanstack/react-query`

  CRITICAL: Do NOT use `apiFetch` for the history fetch. The ingestion router prefix is `/ingestion` (not `/api/ingestion`). This is a confirmed prior decision from Phase 9. `apiFetch` prepends `/api` which would result in `/api/ingestion/history` — a non-existent endpoint.
  </action>
