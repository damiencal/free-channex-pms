  <files>frontend/src/components/actions/RVshareEntryForm.tsx</files>
  <action>
  Create `frontend/src/components/actions/RVshareEntryForm.tsx` — a collapsible inline form for manual RVshare booking entry.

  **Behavior (from CONTEXT.md decisions):**
  - "Add RVshare Booking" `Button` (variant="outline", size="sm") expands the form inline via `Collapsible` component
  - Collapsed by default to keep page clean
  - Property pre-selected from AppShell header selector (read from `usePropertyStore`)
  - After successful submit: show "Booking added" inline success message, then collapse form back
  - Validate on blur (field exit) — immediate per-field feedback

  **Form fields** (match `RVshareEntryRequest` in `app/ingestion/schemas.py`):
  1. `confirmation_code` (string) — `Input` component, required
  2. `guest_name` (string) — `Input` component, required
  3. `check_in_date` (date) — `<input type="date">` via `Input` component, required
  4. `check_out_date` (date) — `<input type="date">` via `Input` component, required
  5. `net_amount` (decimal) — `Input` with `type="number"` and `step="0.01"`, required
  6. `property_slug` (string) — `Select` component populated from `['dashboard', 'properties']` query cache, required
  7. `notes` (string | null) — `<textarea>` styled to match Input component, optional

  **Property slug resolution:**
  - Fetch properties from the TanStack Query cache: `useQuery<Property[]>({ queryKey: ['dashboard', 'properties'] })` where `Property` has `{ id: number, slug: string, name: string }`. This query is already cached by AppShell's Header component.
  - Pre-select the property matching `usePropertyStore().selectedPropertyId` — find the property by `id` and use its `slug`
  - Allow user to change property via the Select dropdown

  **Validate on blur (per-field):**
  - Track errors in `Record<string, string>` state (field name -> error message)
  - On blur of each required field: if empty, set error "Required"
  - On blur of `check_out_date`: if `check_out_date <= check_in_date`, set error "Must be after check-in date"
  - On blur of `net_amount`: if `<= 0` or not a valid number, set error "Must be a positive number"
  - Display error text below each field in red (`text-destructive text-xs`)
  - Clear error for a field when user starts typing (on change)

  **Submission:**
  - Use raw `fetch('/ingestion/rvshare/entry', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(data) })` — NOT apiFetch (ingestion endpoints are not under /api/)
  - Wrap in `useMutation` from TanStack Query for loading/error state
  - On submit click: validate ALL fields first (run blur validation on every field). If any errors, abort submit and show errors.
  - On success:
    - Set `submitted = true` which shows "Booking added" message
    - Invalidate `['dashboard']` and `['ingestion', 'history']` query caches
    - After 2 seconds, collapse the form and reset all fields + `submitted` state
  - On error: show error message from backend (`detail` field) below the submit button
  - Submit button shows `Loader2` spinner while pending (same pattern as ActionItem.tsx)

  **Layout:**
  - Form fields in a responsive grid: 2 columns on md+, 1 column on mobile
  - `Label` above each field (use `Label` from `@/components/ui/label`)
  - Submit button at bottom, full width on mobile, auto width on md+
  - "Cancel" text button next to submit to collapse without saving

  **Do NOT:**
  - Install react-hook-form or any form library — useState per field is sufficient for 7 fields
  - Add platform field — platform is always "rvshare", hardcoded in the backend
  - Use apiFetch — ingestion endpoints are at /ingestion/, not /api/ingestion/

  Create `frontend/src/components/actions/ImportHistoryAccordion.tsx` — a collapsible section showing past import runs.

  **Behavior (from CONTEXT.md decisions):**
  - Collapsible section below the upload area — collapsed by default
  - Show last 10 imports by default with "Show more" link for older entries
  - Columns: timestamp, platform, original filename, record count, success/error status
  - Note: Import history endpoint does NOT return property — omit that column (ImportRun has no property FK; single import can span multiple properties)
  - Reverse-chronological list (backend already sorts this way)

  **Implementation:**
  - Use `Collapsible`, `CollapsibleTrigger`, `CollapsibleContent` from `@/components/ui/collapsible`
  - Trigger: a `Button` variant="ghost" with `ChevronDown` icon that rotates when open, text "Import History"
  - Use `useImportHistory(limit)` hook from `@/hooks/useImportHistory`
  - `limit` state: start at 10, "Show more" sets to 50
  - Loading: show `Skeleton` placeholder rows (same pattern as ActionsTab)
  - Error: show inline error text
  - Empty: show muted text "No imports yet"

  **History item display:**
  - Render as a compact list/table inside the collapsible content
  - Each row shows:
    - `imported_at` formatted as locale date+time (use `new Date(run.imported_at).toLocaleString()`)
    - `platform` with capitalize (e.g., "Airbnb", "VRBO", "Mercury")
    - `filename` (truncate with ellipsis if very long — `truncate` Tailwind class, `max-w-[200px]`)
    - Record count: show `inserted + updated` total, or break down as "N new, M updated" if both > 0
  - Use alternating row backgrounds for readability or border-b separator
  - Responsive: on mobile, stack fields vertically per row; on md+ use a horizontal row layout

  **"Show more" link:**
  - Only visible when `data.length === limit` (implies there may be more)
  - When clicked: set limit to 50 (TanStack Query auto-refetches due to queryKey change)
  - If already showing 50, hide the "Show more" link
