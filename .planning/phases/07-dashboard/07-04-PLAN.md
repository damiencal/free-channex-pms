    **useBookings hook** (frontend/src/hooks/useBookings.ts):
    - `useBookings(startDate: string, endDate: string)`: TanStack Query hook calling GET /api/dashboard/bookings with selectedPropertyId and date range
    - Query key includes selectedPropertyId from Zustand store so data refetches on property change
    - Returns typed array of Booking objects: { id, platform, guest_name, check_in_date, check_out_date, net_amount, property_slug, property_display_name }
    - staleTime: 5 minutes

    **BookingBar** (frontend/src/components/calendar/BookingBar.tsx):
    - Renders a horizontal bar representing a booking
    - Props: booking object, onClick callback, startCol (1-7), span (number of day cells to span)
    - Background color from getPlatformColor(booking.platform) — use CSS custom properties so dark mode works automatically
    - Shows guest name (truncated if too long) inside the bar
    - Rounded corners on the start/end of the booking (but not on week-boundary splits)
    - When a booking spans across a week boundary, render as two separate bars: one ending Saturday, one starting Sunday. The parent MonthCalendar handles this splitting logic.
    - Cursor pointer on hover

    **BookingPopover** (frontend/src/components/calendar/BookingPopover.tsx):
    - Uses shadcn Popover component (NOT Tooltip — per RESEARCH.md, avoid nesting Tooltip+Popover)
    - Trigger: the BookingBar component
    - Content: compact detail card showing:
      - Guest name (bold)
      - Dates: "Feb 15 - Feb 20" format with night count "(5 nights)"
      - Platform: with platform color dot
      - Amount: formatted as currency
      - Property: display name
    - Popover closes on click outside (default shadcn behavior)

    **MonthCalendar** (frontend/src/components/calendar/MonthCalendar.tsx):
    - 7-column CSS Grid layout
    - Header row: Sun, Mon, Tue, Wed, Thu, Fri, Sat
    - Day cells: numbered 1-28/29/30/31 with leading/trailing empty cells for proper day-of-week alignment
    - Booking bars positioned within the grid:
      - For each booking overlapping the current month, calculate which day cells it occupies
      - Split bookings at week boundaries (Saturday->Sunday) into separate BookingBar segments
      - Stack overlapping bookings vertically within day cells (if multiple bookings on same days)
      - Each day cell has a min-height to accommodate stacked bookings
    - Month navigation: left/right arrow buttons to move month, month/year title in center
    - Current day highlighted with a subtle ring/border
    - Uses the booking data from useBookings() with the date range covering the displayed month (plus a few days buffer for partial-month bookings)

    **CalendarTab** (frontend/src/components/calendar/CalendarTab.tsx):
    - State: currentMonth (Date), viewMode ('month' | 'timeline')
    - View toggle: two buttons (or shadcn Tabs sub-tabs) to switch between Month and Timeline views
    - Passes currentMonth and bookings to whichever view is active
    - Timeline view renders a placeholder for now (Task 2)
    - Loading state: show skeleton grid matching calendar dimensions
    - Error state: show ErrorAlert with retry
