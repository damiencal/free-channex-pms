<task type="auto">
  <name>Task 1: Create dashboard API router with properties and metrics endpoints</name>
  <files>app/api/dashboard.py</files>
  <action>
    Create app/api/dashboard.py with a router prefixed at `/api/dashboard` and tagged `["dashboard"]`.

    **GET /api/dashboard/properties**
    - Query all Property rows, return list of {id, slug, display_name}
    - No filtering needed (always returns all properties)

    **GET /api/dashboard/metrics?property_id=optional**
    - Returns a JSON object with:
      - `ytd_revenue`: Sum of revenue journal lines for current year (Jan 1 through today), as string with 2 decimals
      - `ytd_expenses`: Sum of expense journal lines for current year, as string with 2 decimals
      - `current_month_profit`: Revenue minus expenses for current calendar month, as string with 2 decimals
      - `yoy_revenue_change`: Percentage change vs same period last year (e.g., "+12.5%" or "-3.2%"), null if no prior year data
      - `yoy_expense_change`: Same for expenses
      - `actions_count`: Count of pending actions (from actions endpoint logic, integer)
    - Filter by property_id if provided (filter on JournalEntry.property_id)
    - Use existing JournalLine/JournalEntry/Account models with the signed amount convention (positive=debit, negative=credit):
      - Revenue = accounts with type "revenue" -> sum of journal lines (will be negative because credits) -> negate for display
      - Expenses = accounts with type "expense" -> sum of journal lines (will be positive because debits) -> use as-is
    - For YoY: compute same date range shifted back 1 year. If prior year sum is zero, return null for change percentage.

    **GET /api/dashboard/bookings?property_id=optional&start_date=optional&end_date=optional**
    - Returns list of bookings with: id, platform, platform_booking_id, guest_name, check_in_date, check_out_date, net_amount (as string), property_slug, property_display_name
    - Join Property table to get slug and display_name
    - Filter by property_id if provided
    - Filter by date range if provided (bookings where check_in_date <= end_date AND check_out_date >= start_date, i.e., overlapping)
    - Default: no date filter (returns all bookings)
    - Order by check_in_date desc
    - Limit 500 (calendar needs all visible bookings, not paginated)

    **GET /api/dashboard/occupancy?property_id=optional**
    - Returns per-property occupancy data for the last 12 months
    - Structure: list of {property_slug, property_display_name, months: [{year, month, occupied_nights, total_nights, occupancy_rate}]}
    - Occupancy rate = occupied_nights / total_nights in that month
    - Occupied nights = sum of (min(check_out, month_end) - max(check_in, month_start)).days for each booking overlapping that month
    - Compute server-side (not in SQL) for clarity: query all bookings for last 12 months, then compute per-month occupancy in Python
    - Filter by property_id if provided

    **GET /api/dashboard/actions?property_id=optional**
    - Returns list of pending action items from three sources:
      1. **Resort submissions**: ResortSubmission with status='pending' and is_urgent=True OR check_in_date within 7 days. Include: type="resort_form", booking_id, guest_name, check_in_date, property_slug, urgency (days until check-in), submission_id
      2. **VRBO/RVshare messages**: CommunicationLog with status='pending' and message_type in ('welcome', 'pre_arrival'). Include: type="vrbo_message", booking_id, guest_name, message_type, scheduled_for, property_slug, log_id
      3. **Unreconciled transactions**: Bookings with reconciliation_status='unmatched' and net_amount > 0 (only actual revenue bookings, not fee-only rows). Include: type="unreconciled", booking_id, platform, guest_name, net_amount, check_in_date, property_slug
    - Sort combined list by urgency: resort forms first (by days to check-in ASC), then messages (by scheduled_for ASC), then unreconciled (by check_in_date DESC)
    - Filter by property_id if provided (join through Booking.property_id for all three)
    - Return total count alongside the list for badge display

    All monetary amounts returned as strings (matching existing report endpoint convention). All dates returned as ISO format strings.
