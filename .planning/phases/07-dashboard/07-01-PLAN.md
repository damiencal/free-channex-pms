  <action>
    Update Dockerfile to add a Node.js build stage that compiles the frontend, then copies the dist/ output into the Python image.

    Dockerfile changes (multi-stage):
    1. Add a first stage: `FROM node:22-alpine AS frontend-build`
       - WORKDIR /frontend
       - COPY frontend/package.json frontend/package-lock.json ./
       - RUN npm ci
       - COPY frontend/ ./
       - RUN npm run build
    2. In the existing Python stage (rename to `FROM python:3.12-slim AS app`):
       - After existing COPY commands, add: `COPY --from=frontend-build /frontend/dist ./frontend/dist`

    Update app/main.py:
    1. Add CORSMiddleware for local development:
       ```python
       from fastapi.middleware.cors import CORSMiddleware
       app.add_middleware(
           CORSMiddleware,
           allow_origins=["http://localhost:5173"],  # Vite dev server
           allow_methods=["*"],
           allow_headers=["*"],
       )
       ```
    2. Create a custom SPAStaticFiles class that serves index.html for any non-file path (SPA fallback):
       ```python
       from fastapi.staticfiles import StaticFiles
       from starlette.staticfiles import StaticFiles as StarletteStaticFiles
       import os

       class SPAStaticFiles(StarletteStaticFiles):
           async def get_response(self, path, scope):
               try:
                   return await super().get_response(path, scope)
               except Exception:
                   return await super().get_response("index.html", scope)
       ```
    3. Mount AFTER all API routers (MUST be last): `app.mount("/", SPAStaticFiles(directory="frontend/dist", html=True), name="spa")`
    4. Guard the mount with `if os.path.isdir("frontend/dist"):` so the app still starts without the frontend build (e.g., during backend-only dev).

    docker-compose.yml: No changes needed (the app service build context already covers the whole repo).
  </action>

  </verify>
