**Dockerfile** — based on RESEARCH.md verified pattern:
```dockerfile
FROM python:3.12-slim

COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

WORKDIR /app

# Install dependencies (cached layer — copy lock files first)
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-cache

# Copy application code (NOT config/ or templates/ — those are bind-mounted)
COPY app/ ./app/
COPY alembic/ ./alembic/
COPY alembic.ini ./
COPY manage.py ./

# Default CMD (overridden by docker-compose command)
CMD ["/app/.venv/bin/uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Do NOT COPY config/ or templates/ in the Dockerfile — they are bind-mounted from the host so operators can edit them without rebuilding. This is a locked decision from CONTEXT.md (config changes require restart, not rebuild).

**docker-compose.yml** — based on RESEARCH.md verified pattern:
```yaml
services:
  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

  app:
    build: .
    restart: unless-stopped
    command: >
      sh -c "/app/.venv/bin/alembic upgrade head &&
             /app/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000"
    env_file: .env
    ports:
      - "8000:8000"
    volumes:
      - ./config:/app/config:ro
      - ./templates:/app/templates:ro
      - ./pdf_mappings:/app/pdf_mappings:ro
    depends_on:
      db:
        condition: service_healthy

volumes:
  db_data:
```

Key details:
- Alembic runs BEFORE uvicorn in the command (not in FastAPI lifespan — anti-pattern from research)
- Use `/app/.venv/bin/alembic` and `/app/.venv/bin/uvicorn` since uv creates a venv
- config/, templates/, pdf_mappings/ are bind-mounted read-only (:ro)
- db uses named volume `db_data` for persistence (INFR-06)
- db has healthcheck with `pg_isready` — app waits for healthy DB before starting
- Add `extra_hosts: ["host.docker.internal:host-gateway"]` to app service for Ollama connectivity from inside Docker (Linux compatibility — Mac/Windows get this automatically)

**.env.example** — document every required env var:

POSTGRES_USER=rental

Note: DATABASE_URL uses `+psycopg` (not `+psycopg3`) — this is the correct SQLAlchemy dialect for psycopg3 (pitfall from RESEARCH.md).
