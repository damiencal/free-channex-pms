**AppShell.tsx print modifications:**

Add `print:hidden` to the Header and main TabsList so only the active report content prints (see 12-RESEARCH.md Pattern 7 and Pitfall 6).

1. Wrap the `<Header />` call with a div that has `print:hidden`:
   ```tsx
   <div className="print:hidden">
     <Header />
   </div>
   ```
   This is the non-invasive approach from the research — does not require modifying Header.tsx props.

2. Add `print:hidden` to the TabsList wrapper div:
   ```tsx
   <div className="mb-6 overflow-x-auto print:hidden">
     <TabsList>...</TabsList>
   </div>
   ```
   The TabsContent for inactive tabs is already display:none by Radix, so they won't print. Only the active tab's content will appear.

**IMPORTANT:** Do NOT add print:hidden to the `<main>` element or any `<TabsContent>` wrapper — that would hide the report itself. Only hide the Header and the TabsList navigation bar.

**index.css print rules:**

Add a print media query for layout normalization. Add this at the end of the file or within an appropriate @layer:

```css
@media print {
  body {
    background: white !important;
    color: black !important;
  }

  /* Remove padding from main content area for full-width print */
  main {
    padding: 0 !important;
  }

  /* Ensure collapsible sections don't break across pages awkwardly */
  [data-slot="collapsible"] {
    break-inside: avoid;
  }

  /* Tables should not break mid-row */
  tr {
    break-inside: avoid;
  }
}
```

Also add `print:hidden` to the Reports sub-tab TabsList within ReportsTab.tsx — the report sub-tabs (P&L | Balance Sheet | Income Statement) should also be hidden on print since only the active report matters:

Read `frontend/src/components/reports/ReportsTab.tsx` and add `print:hidden` to the `<div className="overflow-x-auto">` wrapper around the sub-tab TabsList:
```tsx
<div className="overflow-x-auto print:hidden">
  <TabsList>...</TabsList>
</div>
```
