Create `frontend/src/api/reports.ts` with:

**Types (all exported):**
- `PLParams`: { start_date?, end_date?, month?, quarter?, year?, ytd?, breakdown?: 'combined' | 'property' }
- `BalanceSheetParams`: { as_of: string } (ISO date)
- `IncomeStatementParams`: { start_date?, end_date?, month?, quarter?, year?, ytd?, breakdown?: 'totals' | 'monthly' }
- `PLCombinedResponse`: { period: { start_date, end_date }, breakdown: 'combined', revenue: { by_platform: Record<string, { months: Array<{ month, amount }>, subtotal }>, total }, expenses: { by_category: Record<string, string>, total }, net_income }
- `BalanceSheetResponse`: { as_of, assets: { accounts: Array<{ number?, name, balance }>, total }, liabilities: { accounts: Array<{ number?, name, balance }>, total }, equity: { accounts: Array<{ name, balance }>, total }, total_liabilities_and_equity }
- `IncomeStatementTotalsResponse`: { period, breakdown: 'totals', revenue: { by_account: Record<string,string>, total }, expenses: { by_account: Record<string,string>, total }, net_income }
- `IncomeStatementMonthlyResponse`: { period, breakdown: 'monthly', months: Array<{ month, revenue: { by_account, total }, expenses: { by_account, total }, net_income }>, totals: { revenue: { by_account, total }, expenses: { by_account, total }, net_income } }

**Fetch functions:**
- `fetchPL(params: PLParams): Promise<PLCombinedResponse>` — builds URLSearchParams from non-null fields, defaults breakdown to 'combined'. Uses `apiFetch<PLCombinedResponse>(`/reports/pl?${qs}`)`.
- `fetchBalanceSheet(params: BalanceSheetParams): Promise<BalanceSheetResponse>` — passes `as_of` param. Uses `apiFetch`.
- `fetchIncomeStatement(params: IncomeStatementParams): Promise<IncomeStatementTotalsResponse | IncomeStatementMonthlyResponse>` — builds URLSearchParams, defaults breakdown to 'totals'. Uses `apiFetch`.

Follow the exact pattern from `src/api/finance.ts` — import `apiFetch` from `@/api/client`, build query strings with URLSearchParams, set params conditionally.

Create `frontend/src/hooks/useReports.ts` with:

Three hooks following the manual-fetch pattern (see 12-RESEARCH.md Pattern 2):
- `usePL()`: useState<PLParams | null>(null), useQuery with queryKey ['reports', 'pl', params], queryFn calling fetchPL(params!), enabled: params !== null, staleTime: 5 * 60 * 1000. Returns { ...query, generate: setParams, hasGenerated: params !== null }.
- `useBalanceSheet()`: Same pattern with BalanceSheetParams, queryKey ['reports', 'balance-sheet', params], fetchBalanceSheet.
- `useIncomeStatement()`: Same pattern with IncomeStatementParams, queryKey ['reports', 'income-statement', params], fetchIncomeStatement.

All three hooks return `{ ...query, generate: setParams, hasGenerated: params !== null }` — the `generate` function is what the Generate button calls. `hasGenerated` distinguishes "never generated" from "generated but no data".

Create `frontend/src/components/reports/ReportFilters.tsx`:

Shared filter controls component. Props:
- `mode: 'range' | 'snapshot'` — 'range' for P&L and Income Statement (start_date + end_date), 'snapshot' for Balance Sheet (as_of only)
- `onGenerate: (params: { start_date: string; end_date: string } | { as_of: string }) => void` — called when Generate clicked
- `isFetching: boolean` — disables Generate button during fetch
- `className?: string`

Internal state:
- `activePreset: 'this-month' | 'this-quarter' | 'ytd' | 'last-year' | 'custom' | null` — starts null
- `customStart: string` and `customEnd: string` — for custom date range inputs (type="date")

Preset computation (all client-side, no library):
- 'this-month': start = first of current month, end = last day of current month
- 'this-quarter': start = first of current quarter, end = last of current quarter
- 'ytd': start = Jan 1 of current year, end = today
- 'last-year': start = Jan 1 of last year, end = Dec 31 of last year

Layout:
- Row of 4 Button components (variant="outline" by default, variant="default" when active): "This Month", "This Quarter", "YTD", "Last Year"
- Clicking a preset sets activePreset and clears custom inputs
- A "Custom" button that when active reveals two date Input fields (Start Date, End Date)
- For snapshot mode: same presets but they compute an end-of-period `as_of` date. Custom mode shows a single date Input.
- "Generate" Button (variant="default", disabled when isFetching or no preset/custom selected) at the end of the row
- Generate button text: isFetching ? "Generating..." : "Generate"

When Generate is clicked:
- If mode='range': compute { start_date, end_date } from preset or custom and call onGenerate
- If mode='snapshot': compute { as_of } (end date of preset period or custom date) and call onGenerate

Use flex-wrap gap-2 for the button row so it wraps on mobile. Date inputs use existing `Input` component from `@/components/ui/input`.
