Build the normalizer module — the core orchestration layer that ties adapters to the database. It handles: reading CSV bytes with Polars, calling adapter validate/parse functions, archiving raw files before DB writes, upserting records with insert-vs-update detection via PostgreSQL xmax, recording ImportRun entries, and resolving listing identifiers to property IDs via the listing_slug_map config.

Purpose: This is the single pipeline through which all CSV imports flow. Adapters produce records; the normalizer writes them to the database and archives the raw file. Without this, adapters have nowhere to send their output.

  <action>
    Create app/ingestion/normalizer.py implementing the full ingestion pipeline. The module has three public functions:

    **1. `ingest_csv(raw_bytes, filename, platform, adapter, db) -> dict`**

    This is the main CSV ingestion pipeline used by Airbnb, VRBO upload endpoints:

    a. Read CSV with Polars:
       ```python
       import io
       import polars as pl
       df = pl.read_csv(
           io.BytesIO(raw_bytes),
           infer_schema_length=0,    # all columns as Utf8
           null_values=["", "N/A"],
           try_parse_dates=False,
       )
       ```

    b. Call `adapter.validate_headers(df)` — raises ValueError if headers wrong. Let it propagate.

    c. Call `adapter.parse(df)` — returns `(list[BookingRecord], list[str])` where the second element is validation errors. If errors is non-empty, raise ValueError with all errors joined by newlines.

    d. Resolve property_id for each BookingRecord:
       - Build a unified listing_slug_map by iterating all properties from `get_config().properties` and collecting each property's `listing_slug_map` dict. Result is a flat dict: `{listing_identifier: property_slug, ...}`.
       - For each record, look up `record.property_slug` — it should already be set by the adapter (the adapter resolves the listing name from the CSV row using the listing_slug_map). But the normalizer must resolve `property_slug` to `property_id` (integer FK).
       - Query the DB for Property by slug to get the id. Cache this lookup (there are only 2 properties). If slug not found, add to errors list and fail the whole import.

    e. Archive raw file BEFORE any DB writes:
       ```python
       archive_path = archive_file(raw_bytes, filename, platform, config.archive_dir)
       ```
       Archive function writes bytes to `{archive_dir}/{platform}/YYYY-MM-DD_HH-MM-SS_{filename}`.
       Creates directory if it doesn't exist. Uses `datetime.now(timezone.utc)` for timestamp.

    f. Upsert each booking record using PostgreSQL-specific insert:
       ```python
       from sqlalchemy.dialects.postgresql import insert as pg_insert
       from sqlalchemy import text, func
       ```
       For each BookingRecord:
       - Build values dict with property_id (resolved from slug), all fields from record
       - `pg_insert(Booking).values(**values_dict)`
       - `.on_conflict_do_update(index_elements=["platform", "platform_booking_id"], set_={...})`
       - Include `updated_at: func.now()` in set_ (ORM onupdate not triggered by upserts)
       - `.returning(text("xmax"))` to detect insert vs update (xmax == 0 means insert, > 0 means update)
       - Collect inserted_ids and updated_ids lists

    g. Commit the transaction: `db.commit()`

    h. Record ImportRun:
       ```python
       run = ImportRun(
           platform=platform,
           filename=filename,
           archive_path=str(archive_path),
           inserted_count=len(inserted_ids),
           updated_count=len(updated_ids),
           skipped_count=0,
       )
       db.add(run)
       db.commit()
       ```

    i. Return result dict:
       ```python
       return {
           "platform": platform,
           "filename": filename,
           "inserted": len(inserted_ids),
           "updated": len(updated_ids),
           "skipped": 0,
           "inserted_ids": inserted_ids,
           "updated_ids": updated_ids,
       }
       ```

    **2. `ingest_bank_csv(raw_bytes, filename, adapter, db) -> dict`**

    Similar to ingest_csv but for Mercury bank transactions:
    - Same read/validate/parse/archive flow
    - Upserts to BankTransaction table instead of Booking
    - Conflict target is `transaction_id` (single column unique)
    - No property_id resolution needed
    - Records ImportRun with platform="mercury"

    **3. `create_manual_booking(entry: RVshareEntryRequest, db) -> dict`**

    For manual RVshare bookings (no CSV involved):
    - Resolve property_slug to property_id (query Property table by slug, raise ValueError if not found)
    - Upsert to Booking table with platform="rvshare", platform_booking_id=entry.confirmation_code
    - Record ImportRun with platform="rvshare", filename="manual_entry", archive_path="N/A"
    - Return result dict with insert/update status

    **Private helper functions:**

    - `archive_file(raw_bytes, filename, platform, archive_dir) -> Path` — writes bytes to timestamped path
    - `resolve_property_id(slug, db) -> int` — queries Property table, caches result, raises ValueError if not found
    - `build_listing_lookup() -> dict[str, str]` — builds flat listing_identifier -> property_slug map from all PropertyConfigs

    **Critical implementation notes from CONTEXT.md and RESEARCH.md:**
    - Archive BEFORE DB writes (even failed imports leave a trace)
    - Halt-and-report: validate everything before writing anything
    - Explicitly set updated_at in on_conflict_do_update set_ dict
    - Use infer_schema_length=0 for all Polars read_csv calls
    - Use try_parse_dates=False — adapters handle date parsing
