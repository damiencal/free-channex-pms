**generate_balance_sheet(db, as_of_date) -> dict:**

1. **Query account balances up to as_of_date:**
   - Join JournalLine -> JournalEntry -> Account
   - Filter: `JournalEntry.entry_date <= as_of_date` AND `Account.is_active == True`
   - Group by: Account.id, Account.number, Account.name, Account.account_type
   - Select: `func.coalesce(func.sum(JournalLine.amount), Decimal("0"))` as balance

2. **Organize by account type:**
   - Assets (account_type == "asset"): show balance as-is (debits are positive)
   - Liabilities (account_type == "liability"): NEGATE for display (credits stored as negative, show positive)
   - Equity (account_type == "equity"): NEGATE for display

3. **CRITICAL: Loan liability balances:**
   - For loan liability accounts (number 2100, 2110 -- any account linked to a Loan record), do NOT use journal line sums.
   - Instead, query Loan records and call `get_loan_balance(db, loan)` for each.
   - This is because Phase 3 never created loan origination journal entries -- only payment entries exist.
   - Implementation: query all Loan records, call `get_loan_balance()` for each, use those values instead of the journal line sums for those specific accounts.
   - Import `get_loan_balance` from `app.accounting.loans` and `Loan` from `app.models.loan`.

4. **Compute Retained Earnings:**
   - Retained Earnings = cumulative net income from all journal lines up to as_of_date
   - Formula: `-(sum of all revenue lines) - (sum of all expense lines)` where revenue lines are negative (credits) and expense lines are positive (debits)
   - Simpler: query all journal lines where account_type in ('revenue', 'expense') up to as_of_date, sum amounts, negate result
   - This gives: positive if profitable (revenues exceed expenses), negative if unprofitable

5. **Response shape:**
   ```python
   {
       "as_of": "2026-03-31",
       "assets": {
           "accounts": [
               {"number": 1010, "name": "Mercury Checking", "balance": "15000.00"},
               {"number": 1020, "name": "Accounts Receivable", "balance": "0.00"}
           ],
           "total": "15000.00"
       },
       "liabilities": {
           "accounts": [
               {"number": 2010, "name": "Unearned Revenue", "balance": "500.00"},
               {"number": 2100, "name": "RV Purchase Loan Payable", "balance": "25000.00"},
               ...
           ],
           "total": "25500.00"
       },
       "equity": {
           "accounts": [
               {"number": 3000, "name": "Owner Equity", "balance": "0.00"},
               {"name": "Retained Earnings", "balance": "5000.00"}  # computed, not from an account
           ],
           "total": "5000.00"
       },
       "total_liabilities_and_equity": "30500.00"
   }
   ```
   Balance sheet is combined-only -- no per-property breakdown (per CONTEXT.md decision).

**generate_income_statement(db, start_date, end_date, breakdown="totals") -> dict:**

1. **Query revenue and expenses for the period:**
   - Same approach as P&L but simpler: combined totals only (no per-property breakdown)
   - Revenue: sum all journal lines where account_type == "revenue" within date range, group by account name, negate for display
   - Expenses: sum all journal lines where account_type == "expense" within date range, group by account name

2. **Monthly breakdown (breakdown="monthly"):**
   - If breakdown == "monthly", add GROUP BY `extract('year', JournalEntry.entry_date), extract('month', JournalEntry.entry_date)`
   - Return month-by-month rows within the date range
   - Each month has its own revenue/expense/net_income section

3. **Response shape (totals):**
   ```python
   {
       "period": {"start_date": "2026-01-01", "end_date": "2026-12-31"},
       "breakdown": "totals",
       "revenue": {
           "by_account": {
   ```

4. **Response shape (monthly):**

   ```python
           {
   ```

IMPORTANT: The income statement and P&L draw from the same journal line data but are framed differently:
- P&L = per-property focus with platform revenue breakdown (for property management)
- Income statement = combined totals with account-level detail (for tax/accounting)

Sign conventions (critical):
- Revenue accounts: journal lines are NEGATIVE (credits). Negate for display -> positive.
- EXCEPTION: Promotional Discounts is a contra-revenue account (account_type=revenue but debits are positive). It will naturally appear as a positive debit in journal lines. After negation it shows as negative in display, which is correct (it reduces revenue).
- Expense accounts: journal lines are POSITIVE (debits). Show as-is.
- Liability accounts: journal lines are NEGATIVE (credits). Negate for display -> positive.

All monetary amounts as `str(Decimal)` for JSON serialization.
