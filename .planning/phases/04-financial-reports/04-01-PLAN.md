**resolve_period() function:**

```python
from datetime import date

def resolve_period(
    start_date: date | None = None,
    end_date: date | None = None,
    month: int | None = None,
    quarter: str | None = None,  # "Q1", "Q2", "Q3", "Q4"
    year: int | None = None,
    ytd: bool = False,
) -> tuple[date, date]:
```

Resolution priority (first match wins):
1. If `start_date` and `end_date` both provided: return them directly (validate start <= end)
2. If `month` provided (requires `year`): return first and last day of that month
3. If `quarter` provided (requires `year`): parse "Q1"/"Q2"/"Q3"/"Q4" -> return first and last day of that quarter
4. If `year` provided alone: return Jan 1 - Dec 31 of that year
5. If `ytd` is True: return Jan 1 of current year through today
6. If none of the above: raise `ValueError("No valid period specified. Provide start_date/end_date, month+year, quarter+year, year, or ytd=True.")`

For month last-day calculation, use `calendar.monthrange(year, month)[1]`.

Validation:
- If `month` provided but `year` is None, raise `ValueError("month requires year")`
- If `quarter` provided but `year` is None, raise `ValueError("quarter requires year")`
- If `quarter` is not one of "Q1", "Q2", "Q3", "Q4" (case-insensitive), raise `ValueError(f"Invalid quarter: {quarter}. Use Q1, Q2, Q3, or Q4.")`
- If `start_date > end_date`, raise `ValueError("start_date must be on or before end_date")`
