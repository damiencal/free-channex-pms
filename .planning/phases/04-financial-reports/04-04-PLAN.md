<objective>
Create the bank transaction categorization endpoints: list bank transactions with category filter, single-transaction categorization with auto-expense creation, and bulk categorization. These endpoints are added to the existing accounting router since they operate on bank transactions (an accounting concern, not a report).

NOTE: Phase 3 (plan 03-06) already established `/api/accounting/bank-transactions` as the prefix for bank transaction endpoints (GET list endpoint). Categorization endpoints MUST use this same prefix for architectural consistency: `PATCH /api/accounting/bank-transactions/{id}/category` and `PATCH /api/accounting/bank-transactions/categorize`. The CONTEXT.md notation "PATCH /bank-transactions/{id}/category" is relative shorthand, not an absolute path.

Purpose: Bank transaction categorization (DASH-08) allows users to assign categories to imported bank transactions. When an expense category is assigned, the system auto-creates the corresponding expense journal entry so the transaction automatically appears in P&L reports. Non-expense categories (owner deposits, loan payments, transfers, personal) are recorded but do not create journal entries.

Output: Three new endpoints in `app/api/accounting.py`: GET list, PATCH single, PATCH bulk. Full paths: `GET /api/accounting/bank-transactions`, `PATCH /api/accounting/bank-transactions/{id}/category`, `PATCH /api/accounting/bank-transactions/categorize`.

**1. GET /api/accounting/bank-transactions:**

```python
@router.get("/bank-transactions")
def list_bank_transactions(
    categorized: str | None = Query(default=None, description="Filter: 'true', 'false', or 'all' (default)"),
    start_date: date | None = Query(default=None),
    end_date: date | None = Query(default=None),
    limit: int = Query(default=100, ge=1, le=1000),
    offset: int = Query(default=0, ge=0),
    db: Session = Depends(get_db),
) -> list[BankTransactionResponse]:
```

Implementation:
- Build a SELECT on BankTransaction, ordered by date DESC, id DESC
- If `categorized == "true"`: filter `BankTransaction.category.is_not(None)`
- If `categorized == "false"`: filter `BankTransaction.category.is_(None)`
- If `categorized` is None or "all": no category filter
- Apply start_date/end_date filters on `BankTransaction.date`
- Apply limit/offset
- Return list of BankTransactionResponse

**2. PATCH /api/accounting/bank-transactions/{txn_id}/category:**

```python
@router.patch("/bank-transactions/{txn_id}/category")
def categorize_single_transaction(
    txn_id: int,
    body: SingleCategoryRequest,
    db: Session = Depends(get_db),
) -> BankTransactionResponse:
```

Implementation:
1. Look up BankTransaction by txn_id. 404 if not found.
2. Validate `body.category` is in `ALL_CATEGORIES`. 422 if invalid with message listing valid categories.
3. If the transaction already has a `journal_entry_id` (was previously categorized with an expense category and has an auto-created journal entry): return 422 with "Transaction already has a journal entry. Remove the existing categorization before re-categorizing."
4. If `body.category` is in `EXPENSE_CATEGORIES` (it's an expense category):
   - Require `body.attribution` -- return 422 if missing with "attribution is required for expense categories (jay, minnie, or shared)"
   - Call `record_expense()` from `app.accounting.expenses`:
     ```python
     expense = record_expense(
         db=db,
         expense_date=txn.date,
         amount=abs(txn.amount),  # bank transactions are negative for debits, expense amount must be positive
         category=body.category,
         description=txn.description or f"Bank transaction {txn.transaction_id}",
         attribution=body.attribution,
         property_id=body.property_id,
     )
     ```
   - Set `txn.journal_entry_id = expense.journal_entry_id`
5. If `body.category` is in `NON_EXPENSE_CATEGORIES`: just set the category, no journal entry.
6. Set `txn.category = body.category`
7. `db.commit()`
8. Return BankTransactionResponse

**3. PATCH /api/accounting/bank-transactions/categorize:**

```python
@router.patch("/bank-transactions/categorize")
def categorize_bulk_transactions(
    body: BulkCategoryRequest,
    db: Session = Depends(get_db),
) -> dict:
```

Implementation:
1. Process each assignment in `body.assignments`:
   - Look up BankTransaction by assignment.id. If not found, add to errors and continue.
   - Validate category is in ALL_CATEGORIES. If invalid, add to errors and continue.
   - If txn already has journal_entry_id, add to errors and continue.
   - If expense category: require attribution, call record_expense(), set journal_entry_id.
   - If non-expense category: just set category.
   - Set txn.category = assignment.category
   - Increment success count
2. `db.commit()` once at the end (after all assignments processed)
3. Return:
   ```python
   {
       "categorized": success_count,
       "errors": [{"id": txn_id, "error": "message"}, ...]
   }
   ```

IMPORTANT: Use `abs(txn.amount)` when calling record_expense() because bank transaction amounts for debits are negative, but record_expense() requires positive amounts.

IMPORTANT: The bulk endpoint path `/bank-transactions/categorize` must be registered BEFORE the single endpoint `/bank-transactions/{txn_id}/category` in the file, OR use a route that won't conflict. FastAPI matches routes in order -- if `{txn_id}` comes first, "categorize" would be captured as a txn_id. To avoid this, place the bulk endpoint BEFORE the single endpoint in the file. The path `/bank-transactions/{txn_id}/category` has the `/category` suffix so it will NOT conflict with `/bank-transactions/categorize` since they have different path structures. The `{txn_id}` path expects an integer, and "categorize" would fail integer parsing, so FastAPI will route correctly. However, for safety and clarity, place the bulk endpoint first.
