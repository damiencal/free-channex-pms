<objective>
Create the P&L (Profit & Loss) report generator function and its API endpoint. The P&L queries journal lines to compute revenue by platform and expenses by category for any date range, with optional per-property breakdown that correctly allocates shared expenses.

Purpose: The P&L is the primary financial report for the rental business. It shows whether properties are profitable, which platforms generate the most revenue, and where expenses are concentrated. This is the core report that satisfies ACCT-02 (P&L statements per property and combined).

Output: `generate_pl()` function in `app/accounting/reports.py`, `GET /api/reports/pl` endpoint in `app/api/reports.py`, and reports router registered in `app/main.py`.

**Function signature:**
```python
def generate_pl(
    db: Session,
    start_date: date,
    end_date: date,
    breakdown: str = "combined",  # "combined" or "property"
) -> dict:
```

**Implementation approach:**

1. **Query revenue from journal lines:**
   - Join JournalLine -> JournalEntry -> Account
   - Filter: `Account.account_type == "revenue"` AND `JournalEntry.entry_date BETWEEN start_date AND end_date`
   - Also filter: `JournalEntry.source_type == "booking_payout"` for revenue lines (this excludes adjustments from non-booking sources)
   - Group by: `JournalEntry.property_id`, extract platform from `JournalEntry.source_id` (the source_id format is `booking_payout:{platform}:{booking_id}`), AND `extract('year', JournalEntry.entry_date), extract('month', JournalEntry.entry_date)` to produce monthly rows per platform
   - Sign convention: Revenue journal lines are NEGATIVE (credits). Negate them for display so revenue shows as positive numbers.

2. **Extract platform from source_id:**
   - `source_id.split(":")[1]` gives the platform name ("airbnb", "vrbo", "rvshare")
   - For non-booking revenue (adjustments, discounts), these will have different source_types and should be excluded from the platform breakdown. Include only `source_type == "booking_payout"` entries.

3. **Query expenses from journal lines:**
   - Join JournalLine -> JournalEntry -> Account
   - Filter: `Account.account_type == "expense"` AND `JournalEntry.entry_date BETWEEN start_date AND end_date`
   - Group by: `Account.name` (category name) and `JournalEntry.property_id`
   - Sign convention: Expense journal lines are POSITIVE (debits). Show as-is.

4. **Query properties for breakdown:**
   - `db.query(Property).all()` to get all active properties with their IDs and display names
   - Active property count = `len(properties)` -- used for shared expense allocation

5. **Build the response dict:**

   If `breakdown == "combined"` (default):
   ```python
   {
       "period": {"start_date": "2026-01-01", "end_date": "2026-12-31"},
       "breakdown": "combined",
       "revenue": {
           "by_platform": {
               "airbnb": {
                   "months": [
                       {"month": "2026-01", "amount": "1500.00"},
                       {"month": "2026-02", "amount": "1800.00"},
                       ...
                   ],
                   "subtotal": "5000.00"
               },
               "vrbo": {
                   "months": [...],
                   "subtotal": "3000.00"
               },
               "rvshare": {
                   "months": [...],
                   "subtotal": "1000.00"
               }
           },
           "total": "9000.00"
       },
       "expenses": {
           "by_category": {
               "Repairs & Maintenance": "500.00",
               "Utilities": "300.00",
               "Platform Fees": "200.00",
               ...
           },
           "total": "1000.00"
       },
       "net_income": "8000.00"  # revenue.total - expenses.total
   }
   ```

   NOTE: Each platform under `by_platform` contains a `months` list (sorted chronologically) with `{"month": "YYYY-MM", "amount": "..."}` entries, plus a `subtotal` for that platform's total across all months. This satisfies the CONTEXT.md requirement: "P&L revenue broken down by platform + month (Airbnb/VRBO/RVshare subtotals, each with monthly rows)".

   If `breakdown == "property"`:
   ```python
   {
       "period": {"start_date": "2026-01-01", "end_date": "2026-12-31"},
       "breakdown": "property",
       "properties": {
           "Jay": {
               "property_id": 1,
               "revenue": {
                   "by_platform": {
                       "airbnb": {
                           "months": [{"month": "2026-01", "amount": "..."}, ...],
                           "subtotal": "3000.00"
                       },
                       "vrbo": {
                           "months": [...],
                           "subtotal": "1000.00"
                       }
                   },
                   "total": "4000.00"
               },
               "expenses": {
                   "by_category": {"Utilities": "200.00", ...},
                   "total": "700.00"  # includes 1/N share of shared expenses
               },
               "net_income": "3300.00"
           },
           "Minnie": { ... },
       },
       "combined": {
           "revenue": {"by_platform": {"airbnb": {"months": [...], "subtotal": "..."}, ...}, "total": "9000.00"},
           "expenses": {"by_category": {...}, "total": "1000.00"},
           "net_income": "8000.00"
       }
   }
   ```

6. **Shared expense allocation (per-property breakdown only):**
   - Property-specific expenses (property_id is set): attributed to that property
   - Shared expenses (property_id is NULL): split evenly across all active properties
   - `allocated_share = shared_expense_amount / property_count`
   - CRITICAL: In the combined totals, use the FULL shared expense amount (do not double-count from per-property allocations)
   - Query property count dynamically from the Property table -- NEVER hardcode to 2

7. **Decimal serialization:**
   - All monetary amounts in the response dict should be `str(amount)` (Decimal -> string) to avoid JSON float issues
   - Use `Decimal("0")` as default for missing platforms/categories

IMPORTANT pitfalls to avoid:
- Do NOT query bookings.net_amount for revenue -- use journal_lines for Rental Income account
- Do NOT include Platform Fees in the revenue section -- they are expenses (account_type == "expense")
- Revenue lines are negative in journal_lines (credits) -- NEGATE for display
- Shared expense in combined P&L = whole amount once; in per-property = 1/N each

Import requirements: `from sqlalchemy import func, select, case`, `from sqlalchemy.orm import Session`, `from app.models.journal_line import JournalLine`, `from app.models.journal_entry import JournalEntry`, `from app.models.account import Account`, `from app.models.property import Property`
