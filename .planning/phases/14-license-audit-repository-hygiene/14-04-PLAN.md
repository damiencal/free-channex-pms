  <name>Task 1: Rewrite git history to remove PII</name>
  <files>(git history rewrite -- no working tree files modified)</files>
  <action>
    **IMPORTANT: This task rewrites git history. It must run AFTER all other Phase 14 plans are complete.**

    **Step 1: Install git-filter-repo**
    ```bash
    brew install git-filter-repo  # or pip install git-filter-repo
    ```

    **Step 2: Create a replacements file for PII scrubbing**

    Create a temporary file `/tmp/pii-replacements.txt` with the following literal text replacements (one per line, format: `ORIGINAL==>REPLACEMENT`):

    ```
    CHANGE_ME==>CHANGE_ME
    CHANGE_ME==>CHANGE_ME
    ```

    Note: The config/jay.yaml and config/minnie.yaml files contain real property addresses but those files are being removed from tracking entirely. git-filter-repo can also remove them from history.

    **Step 3: Run git-filter-repo**

    First, ensure there are no uncommitted changes: `git status` should show clean working tree.

    Then run:
    ```bash
    git filter-repo --replace-text /tmp/pii-replacements.txt --force
    ```

    This rewrites every commit in history, replacing occurrences of "CHANGE_ME" and "CHANGE_ME" with "CHANGE_ME".

    **Step 4: Also remove the real config files from history entirely**

    The config/jay.yaml and config/minnie.yaml had real addresses in past commits. Remove them from all history:
    ```bash
    git filter-repo --path config/jay.yaml --path config/minnie.yaml --invert-paths --force
    ```

    Wait -- git-filter-repo can only be run once on a repo (subsequent runs need `--force` and the repo must be a "fresh clone"). The better approach is to combine both operations.

    Actually, git-filter-repo with `--replace-text` already handles text in those files. But to fully remove the files from history:

    **Combined approach:** Run the text replacement first (catches PII in all files including jay.yaml and minnie.yaml in old commits), then separately remove the files from history if needed. Since the addresses in jay.yaml/minnie.yaml ("CHANGE_ME_ADDRESS/170") are resort addresses (not truly private -- the resort is a public place), the text replacement of personal names is sufficient. The property-specific config files being untracked going forward (from Plan 03) means they won't appear in future commits.

    If the user wants the resort addresses scrubbed from history too, add them to the replacements file:
    ```
    CHANGE_ME_ADDRESS==>CHANGE_ME_ADDRESS
    CHANGE_ME_ADDRESS==>CHANGE_ME_ADDRESS
    ```

    Include these in the replacements file to be safe.

    **Final command:**
    ```bash
    git filter-repo --replace-text /tmp/pii-replacements.txt --force
    ```

    **Step 5: Verify the rewrite**
    ```bash
    git log --all --oneline | wc -l  # Should still have ~244 commits
    git log --all -p | grep -c "CHANGE_ME"  # Should be 0
    git log --all -p | grep -c "CHANGE_ME"  # Should be 0
    ```
  </action>
  <verify>

<task type="auto">
  <name>Task 2: Scan full history with gitleaks and verify clean</name>
  <files>(no files modified -- scan only)</files>
  <action>
    **Step 1: Install gitleaks**
    ```bash
    brew install gitleaks
    ```

    **Step 2: Run full history scan**
    ```bash
    gitleaks detect --source . --verbose --report-path /tmp/gitleaks-report.json
    ```

    This scans every commit in history for:
    - API keys, tokens, passwords
    - Email addresses (personal ones)
    - AWS/GCP/Azure credentials
    - Private keys
    - Other secret patterns

    **Step 3: Review findings**

    If gitleaks reports findings:
    - Check each finding -- is it a real secret or false positive?
    - False positives: example values like "CHANGE_ME", test data, placeholder URLs
    - Real secrets: need additional git-filter-repo pass to remove them
    - Common false positive: `.env.example` files with placeholder values -- these are fine

    If any real secrets are found, create an additional replacements file and run git-filter-repo again.

    **Step 4: Final PII grep**

    Run a broad search for common PII patterns across ALL history:
    ```bash
    git log --all -p | grep -iE "(real.*name|ssn|social.security|credit.card|password\s*[:=]\s*['\"][^C])" | head -20
    ```

    This is a sanity check -- the specific PII items were already handled, but this catches anything unexpected.
  </action>
  <verify>
    1. `gitleaks detect --source . --verbose` exits with code 0 (no leaks found) or all findings are confirmed false positives
    2. `git log --all -p | grep -c "CHANGE_ME"` returns 0
    3. `git log --all -p | grep -c '"CHANGE_ME"'` returns 0
    4. No real API keys, passwords, or secrets found in any commit
  </verify>
  <done>
    Full git history scan is clean. gitleaks reports zero real findings. No PII or secrets exist in any commit. The repository is safe for public exposure.
  </done>
</task>
