  <action>
    **Step 1: Update pyproject.toml**

    - Add `"pypdf>=4.0"` in its place
    - Run `uv sync` to install pypdf and remove pymupdf

    **Step 2: Rewrite app/compliance/pdf_filler.py using pypdf**

    The module must preserve the EXACT same public API: `detect_form_type()`, `list_form_fields()`, and `fill_resort_form()` with identical signatures and return types.

    Replace the implementation using this pattern from the research:

    ```python
    from pypdf import PdfReader, PdfWriter
    from pypdf.constants import AnnotationDictionaryAttributes
    ```

    **detect_form_type(pdf_path: str) -> str:**
    - Use `PdfReader(pdf_path)`
    - Check `reader.get_fields()` -- if None or empty, return "none"
    - Check for XFA: look at `/AcroForm` in `reader.trailer["/Root"]` for `/XFA` key
    - If XFA present, return "xfa"
    - Otherwise return "acroform"

    **list_form_fields(pdf_path: str) -> list[dict]:**
    - Use `PdfReader(pdf_path)`
    - Iterate `reader.get_fields()` to build list of dicts with keys: page, name, type, current_value
    - For field type, map pypdf field types to human-readable strings

    **fill_resort_form(template_pdf_path, mapping_json_path, booking_data, property_data) -> bytes:**
    - Use `PdfReader(template_pdf_path)` and `PdfWriter()`
    - `writer.append(reader)` to clone all pages
    - Build field_values dict from mapping JSON (same logic as current -- booking/property/static sources, date formatting)
    - For each page: `writer.update_page_form_field_values(page, field_values, auto_regenerate=False)`
    - CRITICAL: After filling, flatten form fields to embed appearances permanently. Use `writer.remove_annotations(subtypes="/Widget")` to remove interactive widgets OR iterate pages and set `/Ff` (field flags) to read-only
    - Actually, the research pattern is: `update_page_form_field_values(page, field_values, auto_regenerate=False)` with the flatten parameter if available. Test that the output renders correctly.
    - Write to `io.BytesIO()` and return `.getvalue()`

    **IMPORTANT:** The `auto_regenerate=False` parameter is critical -- setting it to True causes "save dialog" issues in PDF viewers. The research specifically warns about this.

    **Step 3: Verify the import chain is intact**
    - `app/compliance/submission.py` imports `fill_resort_form` from `app/compliance/pdf_filler` -- this import must still work with no changes needed to submission.py since the function signature is identical.

    Update the module docstring to reference pypdf instead of PyMuPDF.
