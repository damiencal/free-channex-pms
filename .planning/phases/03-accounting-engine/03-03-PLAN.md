**Constants:**

EXPENSE_CATEGORIES = [

**Category-to-account mapping:**
Map each category to its chart of accounts name:
- repairs_maintenance -> "Repairs & Maintenance" (5100)
- supplies -> "Supplies" (5110)
- utilities -> "Utilities" (5120)
- non_mortgage_interest -> "Non-Mortgage Interest" (5130)
- owner_reimbursable -> "Owner Reimbursable" (2200 — this is a LIABILITY, not expense; Dr. Owner Reimbursable, Cr. Cash)
- advertising -> "Advertising" (5140)
- travel_transportation -> "Travel & Transportation" (5150)
- professional_services -> "Professional Services" (5160)
- legal -> "Legal" (5170)
- insurance -> "Insurance" (5180)
- resort_lot_rental -> "Resort Lot Rental Fees" (5190)
- cleaning_service -> "Cleaning Service Fees" (5200)

Note: `owner_reimbursable` is special — it debits a liability account (owner paid, company owes them back), not an expense account. The journal entry is: Dr. Owner Reimbursable (liability increase for amount owed to owner), Cr. Mercury Checking (cash decrease). All other categories debit an expense account.

**`record_expense(db, expense_date, amount, category, description, attribution, property_id=None, vendor=None) -> Expense`**
- Validate category is in EXPENSE_CATEGORIES, raise ValueError if not
- Validate attribution is in ('jay', 'minnie', 'shared'), raise ValueError if not
- Validate amount > 0, raise ValueError if not
- Look up the appropriate account for the category
- Generate source_id: `expense:{expense_date.isoformat()}:{uuid4()}` (uuid ensures uniqueness for multiple expenses on same date)
- Create journal entry via create_journal_entry:
  - For regular expenses: Dr. {expense account} (+amount), Cr. Mercury Checking (-amount)
  - For owner_reimbursable: Dr. Owner Reimbursable (+amount), Cr. Mercury Checking (-amount)
  - source_type: "expense"
  - property_id: passed through
- Create Expense record with journal_entry_id linked
- db.flush()
- Return the Expense

**`bulk_import_expenses(db, rows: list[dict]) -> dict`**
- Each row dict has keys: expense_date (str YYYY-MM-DD), amount (str or float), category (str), description (str), attribution (str), property_id (int|None, optional), vendor (str|None, optional)
- Parse expense_date as date, amount as Decimal
- Call record_expense for each valid row
- Collect errors (row number + error message) for invalid rows
- Return {"imported": count, "errors": [{"row": N, "error": str}]}
- Do NOT abort on individual row errors — process all rows, report all errors

CSV column schema for bulk import (document in docstring):
```
expense_date,amount,category,description,attribution,vendor
2026-01-15,125.50,supplies,"Paper towels and cleaning supplies",jay,Costco
2026-01-20,89.00,utilities,"Internet bill January",shared,Spectrum
```

IMPORTANT: Use `from uuid import uuid4` for source_id generation. Use Decimal for all amount handling. Import EXPENSE_CATEGORIES from this module in other files.
