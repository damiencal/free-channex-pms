**`run_reconciliation(db: Session) -> dict`**
The core reconciliation algorithm:

1. Fetch all unreconciled bookings: `db.query(Booking).filter(Booking.reconciliation_status == "unmatched").all()`
2. Fetch all unreconciled positive bank transactions (deposits): `db.query(BankTransaction).filter(BankTransaction.amount > 0, BankTransaction.reconciliation_status == "unmatched").all()`
3. For each deposit, find candidate booking matches:
   - Amount matches exactly: `booking.net_amount == deposit.amount`
   - Date within window: `abs((deposit.date - booking.check_in_date).days) <= MATCH_WINDOW_DAYS`
   - Per CONTEXT.md: Airbnb typically pays out on check-in day, so check_in_date is the reference date
4. If exactly 1 candidate: AUTO-MATCH
   - Create ReconciliationMatch(booking_id=booking.id, bank_transaction_id=deposit.id, status="matched")
   - Update booking.reconciliation_status = "matched"
   - Update deposit.reconciliation_status = "matched"
   - Remove the matched booking from the unreconciled list (so it cannot match another deposit)
5. If > 1 candidate: FLAG AS NEEDS REVIEW
   - Do NOT create a match record yet
   - Update deposit.reconciliation_status = "needs_review"
   - Add deposit.id to needs_review list
6. If 0 candidates: deposit stays unmatched (no action needed)

Return dict:
```python
{
    "auto_matched": [(booking_id, bank_transaction_id), ...],
    "needs_review": [bank_transaction_id, ...],
    "unmatched_payouts": [booking_id, ...],  # bookings still unmatched after algorithm
    "unmatched_deposits": [bank_transaction_id, ...],  # deposits still unmatched (0 candidates)
}
```

Call `db.flush()` after processing (caller commits).

**`confirm_match(db: Session, booking_id: int, bank_transaction_id: int, confirmed_by: str) -> ReconciliationMatch`**
- Create or update ReconciliationMatch with status="confirmed", confirmed_by set
- Update both booking and bank_transaction reconciliation_status to "confirmed"
- Used for operator-confirmed matches (both auto-matched confirmations and manual matches for "needs_review" items)

**`reject_match(db: Session, match_id: int, confirmed_by: str) -> None`**
- Update ReconciliationMatch status to "rejected"
- Reset both booking and bank_transaction reconciliation_status to "unmatched"
- Used when operator determines an auto-match was incorrect

**`get_unreconciled(db: Session) -> dict`**
- Return dict with:
  - `unmatched_payouts`: list of Booking objects where reconciliation_status == "unmatched"
  - `unmatched_deposits`: list of BankTransaction objects where reconciliation_status == "unmatched" and amount > 0
  - `needs_review`: list of BankTransaction objects where reconciliation_status == "needs_review"
- This is the "unreconciled queue" that operators see

IMPORTANT:
- Compare amounts using Decimal equality (both Booking.net_amount and BankTransaction.amount are Numeric/Decimal -- no float comparison issues)
- Date comparison uses Python date arithmetic (timedelta)
- Remove matched bookings from the candidate pool as you iterate (prevents one booking matching two deposits)
- The algorithm runs as a batch operation -- call it after each import cycle or on-demand
