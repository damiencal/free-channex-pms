**Pydantic response schemas (define in this file):**

```python
class JournalLineResponse(BaseModel):
    id: int
    account_name: str
    account_number: int
    amount: Decimal
    description: str | None

class JournalEntryResponse(BaseModel):
    id: int
    entry_date: date
    description: str
    source_type: str
    source_id: str
    property_id: int | None
    created_at: datetime
    lines: list[JournalLineResponse]

class AccountBalanceResponse(BaseModel):
    account_number: int
    account_name: str
    account_type: str
    balance: Decimal  # net balance (sum of all journal lines for this account)

class ExpenseRequest(BaseModel):
    expense_date: date
    amount: Decimal
    category: str
    description: str
    attribution: str  # jay, minnie, shared
    property_id: int | None = None
    vendor: str | None = None

class ExpenseResponse(BaseModel):
    id: int
    expense_date: date
    amount: Decimal
    category: str
    description: str
    attribution: str
    property_id: int | None
    vendor: str | None
    journal_entry_id: int | None

class LoanPaymentRequest(BaseModel):
    loan_id: int
    principal: Decimal
    interest: Decimal
    payment_date: date
    payment_ref: str  # e.g., "2026-01"

class LoanResponse(BaseModel):
    id: int
    name: str
    account_id: int
    original_balance: Decimal
    interest_rate: Decimal
    start_date: date
    current_balance: Decimal  # computed via get_loan_balance

class ReconciliationResultResponse(BaseModel):
    auto_matched: int
    needs_review: int
    unmatched_payouts: int
    unmatched_deposits: int

class UnreconciledResponse(BaseModel):
    unmatched_payouts: list[dict]  # simplified booking dicts
    unmatched_deposits: list[dict]  # simplified bank transaction dicts
    needs_review: list[dict]  # simplified bank transaction dicts

class MatchConfirmRequest(BaseModel):
    booking_id: int
    bank_transaction_id: int
    confirmed_by: str = "operator"
```

**Endpoints:**

1. **GET /api/accounting/journal-entries** — List journal entries with optional filters
   - Query params: `start_date: date | None`, `end_date: date | None`, `source_type: str | None`, `property_id: int | None`, `limit: int = 100`, `offset: int = 0`
   - Join JournalEntry with JournalLine and Account to build full response
   - Order by entry_date DESC, id DESC
   - Return list[JournalEntryResponse]

2. **GET /api/accounting/journal-entries/{entry_id}** — Get single journal entry with lines
   - Return JournalEntryResponse with lines populated
   - 404 if not found

3. **GET /api/accounting/balances** — Account balances
   - For each active account, sum all JournalLine amounts
   - Return list[AccountBalanceResponse]
   - Optional query param: `account_type: str | None` to filter by type

4. **POST /api/accounting/expenses** — Record single expense
   - Body: ExpenseRequest
   - Call record_expense from accounting/expenses.py
   - Catch ValueError and return 422 with error message
   - Return ExpenseResponse with 201 status

5. **POST /api/accounting/expenses/import** — Bulk CSV import of expenses
   - Accept CSV file upload (same pattern as ingestion CSV endpoints: `file: UploadFile`)
   - Parse CSV with column headers: expense_date, amount, category, description, attribution, vendor (optional), property_id (optional)
   - Call bulk_import_expenses
   - Return {"imported": N, "errors": [...]}

6. **GET /api/accounting/expenses** — List expenses
   - Query params: `start_date: date | None`, `end_date: date | None`, `category: str | None`, `attribution: str | None`, `property_id: int | None`, `limit: int = 100`, `offset: int = 0`
   - Return list[ExpenseResponse]

7. **POST /api/accounting/loans/payments** — Record loan payment
   - Body: LoanPaymentRequest
   - Look up Loan by ID, 404 if not found
   - Call record_loan_payment
   - Catch ValueError and return 422
   - Return JournalEntryResponse with 201 status (or 200 with message if idempotent skip)

8. **GET /api/accounting/loans** — List loans with current balances
   - For each Loan, call get_loan_balance to compute current_balance
   - Return list[LoanResponse]

9. **POST /api/accounting/reconciliation/run** — Trigger reconciliation
   - Call run_reconciliation
   - Commit matches
   - Return ReconciliationResultResponse

10. **GET /api/accounting/reconciliation/unreconciled** — Get unreconciled queue
    - Call get_unreconciled
    - Return UnreconciledResponse with simplified dicts (id, date, amount, description for deposits; id, platform, guest_name, check_in_date, net_amount for payouts)

11. **POST /api/accounting/reconciliation/confirm** — Confirm a match
    - Body: MatchConfirmRequest
    - Call confirm_match
    - Return {"status": "confirmed"}

12. **POST /api/accounting/reconciliation/reject/{match_id}** — Reject a match
    - Call reject_match
    - Return {"status": "rejected"}

IMPORTANT patterns from app/api/ingestion.py to follow:
- Use `db: Session = Depends(get_db)` for all endpoints
- Catch ValueError from accounting modules and re-raise as `HTTPException(status_code=422, detail=str(e))`
- Use `from app.db import get_db`
- For CSV upload: `file: UploadFile`, read with `file.file.read()`, parse with `csv.DictReader(io.StringIO(content.decode("utf-8")))`
- All Decimal fields in Pydantic models use `from decimal import Decimal`
