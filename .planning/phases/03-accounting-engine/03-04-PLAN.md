Create `app/accounting/loans.py`:

**`record_loan_payment(db: Session, loan: Loan, principal: Decimal, interest: Decimal, payment_date: date, payment_ref: str) -> JournalEntry | None`**

- Validate principal >= 0 and interest >= 0 and (principal + interest) > 0, raise ValueError otherwise
- total_payment = principal + interest
- Create journal entry via create_journal_entry:
  - Dr. Loan Liability Account (+principal) — debiting a liability reduces it
  - Dr. Non-Mortgage Interest Expense (+interest) — debiting expense increases it
  - Cr. Mercury Checking (-total_payment) — crediting asset reduces it
  - Sum: +principal + interest - total_payment = +principal + interest - (principal + interest) = 0 (balanced)
- source_type: "loan_payment"
- source_id: `loan_payment:{loan.id}:{payment_ref}` (payment_ref is a caller-provided reference like "2026-01" for January payment)
- entry_date: payment_date
- description: f"Loan payment: {loan.name} — {payment_ref}"
- property_id: None (loans are not property-specific per CONTEXT.md — they are shared liabilities)
- Returns the JournalEntry or None if idempotent skip

Account lookups: Use the loan's `account_id` directly for the liability account. Look up "Non-Mortgage Interest" and "Mercury Checking" by name from the accounts table (cache with a module-level dict like revenue.py does).

IMPORTANT:
- The caller (operator or API) provides the principal and interest split from their lender's amortization schedule. This system does NOT compute the split.
- Use Decimal for all amounts. Validate that principal and interest are not negative.
- The `payment_ref` string should be unique per payment per loan (e.g., "2026-01", "2026-02" for monthly payments). Combined with loan.id in the source_id, this ensures idempotency.

**`get_loan_balance(db: Session, loan: Loan) -> Decimal`**
- Calculate current outstanding balance by summing all journal lines that debit the loan's liability account
- Formula: original_balance - sum(principal payments)
- Query: sum of JournalLine.amount where account_id == loan.account_id (each payment posts a positive/debit amount which reduces the liability)
- Return original_balance - total_principal_paid
- If no payments exist, return original_balance
