<task type="auto">
  <name>Task 3: Create v1.0.0 tag and GitHub Release</name>
  <files>No local files modified — git tag and GitHub release creation</files>
  <action>
    Create an annotated git tag v1.0.0 (separate from existing v1.0 tag) and push it:

    ```bash
    git tag -a v1.0.0 -m "v1.0.0 - Initial open source release"
    git push origin v1.0.0
    ```

    IMPORTANT: v1.0 already exists as an annotated tag. v1.0.0 is a NEW, separate tag. Both will coexist.
    IMPORTANT: Do NOT use `--notes-from-tag` — the existing v1.0 tag annotation contains internal planning details.

    Create the GitHub Release:

    ```bash
    gh release create v1.0.0 \
      --title "v1.0.0 - Initial Open Source Release" \
      --notes "$(cat <<'RELEASE_EOF'
    Roost is a self-hosted vacation rental management platform that automates end-to-end rental operations — from booking notification to accounting entry — with zero manual intervention after initial configuration.

    **Highlights:**
    - Automated booking ingestion from Airbnb via email parsing
    - Guest communication with AI-powered natural language responses
    - Resort compliance form auto-filling and submission
    - Bank transaction reconciliation with fuzzy matching
    - Full accounting ledger with automated journal entries
    - React dashboard with real-time booking management
    - Docker-based self-hosted deployment

    See [CHANGELOG.md](CHANGELOG.md) for the complete list of features and changes.
    RELEASE_EOF
    )" \
      --latest
    ```

    Verify the release:
    ```bash
    gh release view v1.0.0
    ```
  </action>
  <verify>
    Run: `gh release view v1.0.0 --json tagName,name,isLatest`
    Expected: tagName=v1.0.0, name contains "Initial Open Source Release", isLatest=true.
    Run: `git tag -l 'v1.0*'` — shows both v1.0 and v1.0.0.
  </verify>
  <done>
    Git tag v1.0.0 exists and is pushed. GitHub Release v1.0.0 is created as the latest release with highlights and link to CHANGELOG.md.
  </done>
