  1. Create `frontend/src/store/useChatStore.ts`:

  **Types:**
  ```typescript
  export interface QueryResult {
    columns: string[]
    rows: Record<string, unknown>[]
  }

  export interface ChatError {
    type: string      // 'sql_invalid' | 'sql_execution' | 'ollama_down' | 'unknown'
    message: string   // user-friendly message
    detail?: string   // technical detail for Thomas
  }

  export interface ChatMessage {
    id: string
    role: 'user' | 'assistant'
    content: string
    sql?: string
    results?: QueryResult
    error?: ChatError
    isStreaming: boolean
  }
  ```

  **Store interface:**
  ```typescript
  interface ChatStore {
    messages: ChatMessage[]
    addUserMessage: (content: string) => string      // returns id
    addAssistantMessage: () => string                 // returns id
    appendToken: (id: string, token: string) => void
    setSql: (id: string, sql: string) => void
    setResults: (id: string, results: QueryResult) => void
    setError: (id: string, error: ChatError) => void
    setDone: (id: string) => void
  }
  ```

  Implementation uses `create<ChatStore>((set) => ({...}))` with NO persist middleware (ephemeral — clears on reload, per CONTEXT.md decision).

  - `addUserMessage`: generates `crypto.randomUUID()`, appends user message, returns id
  - `addAssistantMessage`: generates id, appends assistant message with `content: ''` and `isStreaming: true`, returns id
  - `appendToken`: maps over messages, finds by id, concatenates token to content
  - `setSql`: maps over messages, finds by id, sets sql field
  - `setResults`: maps over messages, finds by id, sets results field
  - `setError`: maps over messages, finds by id, sets error field and isStreaming to false
  - `setDone`: maps over messages, finds by id, sets isStreaming to false

  2. Create `frontend/src/hooks/useChatStream.ts`:

  **Exports:** `useChatStream()` hook that returns `{ sendMessage, isStreaming }`

  **State:** `isStreaming` boolean via useState.

  **sendMessage(message: string):**
  - Get store state: `const store = useChatStore.getState()`
  - Call `store.addUserMessage(message)`
  - Call `const assistantId = store.addAssistantMessage()`
  - Set isStreaming = true
  - Build history from current messages (last 10, map to `{role, content}`)
  - `fetch('/api/query/ask', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({ message, history }) })`
  - If `!response.ok`, set error on assistantId and return
  - Read stream with `response.body!.getReader()` and `TextDecoder`
  - Buffer-based line parsing:
    - Track `currentEvent` string (default 'message')
    - Lines starting with `event: ` set currentEvent
    - Lines starting with `data: ` dispatch based on currentEvent:
      - `token`: `store.appendToken(assistantId, data)`
      - `sql`: `store.setSql(assistantId, data)`
      - `results`: `store.setResults(assistantId, JSON.parse(data))`
      - `error`: `store.setError(assistantId, JSON.parse(data))`
      - `done`: `store.setDone(assistantId)`
    - Reset currentEvent to 'message' after processing data line
    - Keep incomplete lines in buffer (last element of split)
  - Wrap in try/catch: on network error, set error on assistantId
  - Finally: set isStreaming = false

  Important: Use `useChatStore.getState()` (not the hook) inside sendMessage to avoid stale closures. The hook form `useChatStore()` is only for reading state in components.

  Important: The fetch URL must NOT use the apiFetch helper from api/client.ts — apiFetch expects JSON responses and calls response.json(). The SSE stream needs raw response.body access. Use plain `fetch('/api/query/ask', ...)` directly. In dev mode, Vite proxy forwards /api to the backend.
