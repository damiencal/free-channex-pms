  <action>
Create the togglable expense/loan payment form. This is the primary component for this sub-tab.

**Type toggle:**

Use `useState<'expense' | 'loan_payment'>('expense')`. Render two `Button` components at the top: "Expense" (default variant when active, outline when inactive) and "Loan Payment" (same pattern). Switching type clears the success message and resets the form.

**Expense form fields:**

All fields inside a `<form>` with `onSubmit` handler. Use existing `Label`, `Input`, `Select` components from `@/components/ui/`.

1. `expense_date` (required): `<Input type="date" />` — default to today's date (`new Date().toISOString().split('T')[0]`)
2. `amount` (required): `<Input type="number" step="0.01" min="0.01" placeholder="0.00" />` — positive value only
3. `category` (required): `<Select>` with EXPENSE_CATEGORIES only (import from CategorySelect.tsx or define inline). Group display names using `formatCategoryName()`.
4. `description` (required): `<Input placeholder="e.g., Plumber repair kitchen sink" />`
5. `attribution` (required): `<Select>` with three options: "Jay", "Minnie", "Shared". This is the property attribution per CONTEXT.md — every expense must be attributed.
6. `property_id` (optional but recommended): Read `selectedPropertyId` from `usePropertyStore`. If a property is selected, pre-populate. If attribution is "jay" or "minnie", auto-set property_id to match (read properties from TanStack Query cache `['dashboard', 'properties']` — the AppShell Header populates this). If attribution is "shared", set property_id to null.
7. `vendor` (optional): `<Input placeholder="Optional vendor name" />`

For property_id resolution: the properties cache at `['dashboard', 'properties']` contains an array of `{id, slug, display_name}`. Match attribution string to property slug. Use `queryClient.getQueryData(['dashboard', 'properties'])` for synchronous read.

**Loan payment form fields:**

1. `loan_id` (required): `<Select>` populated from `useLoans()` hook. Display format: `{loan.name} - ${formatCurrency(loan.current_balance)} remaining`. Value is `String(loan.id)`.
2. `principal` (required): `<Input type="number" step="0.01" min="0" placeholder="0.00" />`
3. `interest` (required): `<Input type="number" step="0.01" min="0" placeholder="0.00" />`
4. Total display (read-only): Show computed total `principal + interest` formatted as currency. Update reactively.
5. `payment_date` (required): `<Input type="date" />` — default to today
6. `payment_ref` (required): `<Input placeholder="e.g., 2026-03" />` — month reference for idempotency

**Form state management:**

Use individual `useState` for each field (not a single object). This simplifies reset:

```typescript
const [expenseDate, setExpenseDate] = useState(todayString)
const [amount, setAmount] = useState('')
const [category, setCategory] = useState('')
// ... etc
```

Reset function clears all fields back to defaults (date stays as today, strings cleared).

**Submit handlers:**

Expense submit:
- Validate all required fields are filled. If not, show inline error.
- Call `useCreateExpense` mutation with the `ExpenseRequest` body.
- On success: set success message "Expense recorded - ${formatCurrency(amount)} for ${formatCategoryName(category)}", reset form fields.
- On error: show error message from the mutation error.

Loan payment submit:
- Validate all required fields.
- Call `useLoanPayment` mutation with the `LoanPaymentRequest` body.
- On success: check `data.status`:
  - If `"recorded"`: set success message "Payment recorded"
  - If `"skipped"`: set warning message "Already recorded (duplicate payment reference)" — use amber/yellow text instead of green
- On error: show error message.

**Success/error feedback:**

Use inline text below the toggle and above the form:
- Success: `<p className="text-sm text-green-600 dark:text-green-400">{message}</p>`
- Warning (skipped): `<p className="text-sm text-amber-600 dark:text-amber-400">{message}</p>`
- Error: `<p className="text-sm text-red-600 dark:text-red-400">{message}</p>`

The success message persists until the user interacts with the form again (any input change clears it). This follows the CONTEXT.md decision: "Inline success message + form reset after successful submit — supports batch entry sessions."

**Submit button:**

`<Button type="submit" disabled={mutation.isPending}>` with text "Record Expense" or "Record Payment" based on type. Show "Recording..." text when pending.

**Currency helper:**

```typescript
function formatCurrency(value: string | number): string {
  return Number(value).toLocaleString('en-US', { style: 'currency', currency: 'USD' })
}
```
