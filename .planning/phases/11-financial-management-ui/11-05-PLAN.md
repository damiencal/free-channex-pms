**MatchCandidateList.tsx:**

Component that shows candidate bookings for a needs-review deposit. When a needs-review deposit is expanded, this list shows matching payouts that the user can select to confirm.

Props:
- `deposit: UnreconciledDeposit` (the needs-review deposit)
- `unmatchedPayouts: UnreconciledPayout[]` (all unmatched payouts from the unreconciled response)
- `onConfirm: (bookingId: number, bankTransactionId: number) => void`

Client-side candidate derivation (per research — reproduces server algorithm):
```typescript
function getCandidates(deposit: UnreconciledDeposit, payouts: UnreconciledPayout[]): UnreconciledPayout[] {
  const depositAmount = parseFloat(deposit.amount)
  const depositDate = deposit.date ? new Date(deposit.date + 'T00:00:00') : null
  return payouts.filter(payout => {
    const payoutAmount = parseFloat(payout.net_amount)
    if (payoutAmount !== depositAmount) return false
    if (!depositDate || !payout.check_in_date) return true // if dates missing, match on amount only
    const payoutDate = new Date(payout.check_in_date + 'T00:00:00')
    const daysDiff = Math.abs((depositDate.getTime() - payoutDate.getTime()) / (1000 * 60 * 60 * 24))
    return daysDiff <= 7
  })
}
```

Render each candidate as a clickable row:
- Platform badge (Airbnb/VRBO/RVshare)
- Guest name
- Check-in date
- Net amount
- "Select" button that calls `onConfirm(candidate.id, deposit.id)`

If no candidates found: show "No matching bookings found for this deposit."

Styling: contained list with borders between items. Each item has hover state. Compact rows.
