# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-02
**Phases:** 1-13
**Total Plans:** 56

## Overview

Build a self-hosted vacation rental management platform in thirteen phases. Phases 1-8 deliver the core backend and read-only dashboard. Phases 9-12 close gaps identified by the first v1 milestone audit: integration bug fixes, data import UI, financial management UI, and interactive reports. Phase 13 closes integration bugs and tech debt from the second v1 audit. Each phase delivers a coherent, independently verifiable capability.

## Milestone Summary

**Key Decisions:**
- Self-hosted Docker deployment with single `docker-compose up`
- Local Ollama for LLM queries (text-to-SQL, no external API calls)
- Config-driven architecture (YAML for properties, templates, PDF mappings)
- Polars for CSV ingestion (5-25x faster than pandas)
- PyMuPDF for PDF form filling (field.update() + doc.bake() for cross-viewer rendering)
- Double-entry bookkeeping with signed amounts (positive=debit, negative=credit)
- React + Vite + shadcn/ui for frontend with TanStack Query
- SSE streaming for LLM responses with two-phase pipeline (SQL gen + narrative)
- APScheduler 3.x for pre-arrival scheduling and urgency checks

**Issues Resolved:**
- Router prefix mismatch causing 404s on action buttons (Phase 9)
- Airbnb pre-arrival marking 'sent' without operator notification (Phase 9)
- LLM attribution values incorrect for property-specific queries (Phase 9)
- Revenue recognition requiring manual API call after import (Phase 9)
- No frontend data entry UI (Phase 10)
- No financial management UI (Phase 11)
- Reports tab placeholder (Phase 12)
- Property dropdown blank labels in RVshare form (Phase 13)
- Income category 422 errors in bank transaction categorization (Phase 13)

**Technical Debt Carried Forward:**
- 4 config placeholders (CHANGE_ME — operator setup requirement)
- 3 unverified CSV headers (need real exports to verify)
- 3 cache invalidation gaps (low — data eventually refreshes on tab switch)
- 2 stale module docstrings (low — no runtime impact)
- 1 PDF host info design decision (host info from baked template, not dynamic config)
- 1 external platform dependency (Airbnb native messaging one-time setup)
